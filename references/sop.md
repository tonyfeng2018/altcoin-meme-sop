# 山寨币/Meme SOP 参考手册

## 整体流程

```
步骤零：接收信息（Ticker + Chain + 合约地址）
    ↓
步骤一：一票否决排雷（Kill Switch）
    ↓ 通过 → 步骤二：识别资产属性 → 步骤三：打分
    ↓ 否决 → 输出🔴极度高危，终止
    ↓
步骤四：输出信号与投研建议
```

---

## 步骤零：标准信息接收

必须确认已获取：
- 代币名称 (Ticker) & 项目名称
- 所属公链 (Chain)
- 合约地址 (Contract Address) —— 核心锚点

---

## 步骤一：一票否决排雷（Kill Switch）

> ⚠️ 在计算任何看多得分前，**必须先完成排雷**。只要触发以下任意一条，立即终止评估。

### 🔴 红线一：老鼠仓与发币者声誉
- **工具**：`okx-dex-trenches`（Bundle Detection + Dev Reputation）
- **否决条件**：
  - 同一庄家在同一区块买走大量筹码（Bundle Detection捆绑占比高）
  - Dev Reputation极差（有Rug历史）

### 🔴 红线二：流动性枯竭
- **工具**：`okx-dex-token`（Liquidity Pools数据）
- **否决条件**：
  - 核心池子深度极浅
  - 模拟大额交易滑点（Slippage）> 3%

### 🔴 红线三：蜜罐貔貅
- **工具**：GoPlus安全API / 合约分析
- **否决条件**：
  - 合约未开源
  - 包含隐藏Mint（增发）权限
  - 有黑名单功能
  - 无法卖出

### 🔴 红线四：宏观环境熔断
- **工具**：DXY指数 + VIX
- **否决条件**：
  - BTC处于明确日线级别下跌通道
  - VIX > 25
- **原因**：此时山寨无独立行情，暂停看多操作

---

## 步骤二：识别资产属性与模型路由

获取数据：MC（流通市值）与FDV（全流通市值），计算**流通率 = MC / FDV**

| 流通率 | 模型 | 适用场景 |
|--------|------|---------|
| 15% - 80% | **模型A：常规模型** | 强叙事币 / VC币 |
| > 90% | **模型B：高流通/Meme模型** | 纯土狗 / Meme赛道 |

> ⚠️ 流通率 < 15% 需谨慎，可能为预挖矿或极早期项目，数据可靠性低。

## 步骤四：信号与投研建议

### 总分计算
```
总分 = 模块一得分 + 模块二得分
```

### 评分阈值

| 总分 | 信号 | 结论 | 操作建议 |
|------|------|------|---------|
| **≥ 75分**（且未触发核心扣分项） | 🟢 强烈买入 | 强烈建议买入/建仓 | 筹码结构极佳，聪明钱流入+叙事爆发，高胜率做多标准。执行现货买入或合约右侧建仓。 |
| **55-74分** | ⚪ 底仓观望 | 底仓观望/等待突破 | 有一定叙事或数据亮点，但结构存在瑕疵（小额解锁/大盘无力）。可建观察仓跟踪，不符合重仓标准。 |
| **≤ 40分** | 🔴 规避/减仓 | 规避/减仓/潜在做空标的 | 动能完全衰竭，抛压沉重。现货建议清仓；若标的在高位横盘且面临大额解锁/老鼠仓出货，转入【合约做空观察池】。 |

### 核心扣分项（触发即降级）
- 解锁抛压 -15（未来14天大额悬崖解锁）
- CEX资金流 -10（突发巨额净流入）
- 大户控盘 -15（占比>30%）
- 聪明钱砸盘 -20

## 数据获取优先级

1. **合约地址** → 链上数据（okx-dex系列）
2. **BTC.D + TOTAL3** → 判断大盘资金溢出方向
3. **MC/FDV** → 确定模型路由
4. **解锁日历 + CEX资金流** → 判断抛压
5. **Smart Money + Holder** → 判断庄家意图
6. **社交情绪** → 最后确认市场情绪
