# 山寨币/Meme SOP 参考手册（v1.1）

## 整体流程

```
步骤零：接收信息（Ticker + Chain + 合约地址）
    ↓
步骤一：一票否决排雷（Kill Switch）
    ↓ 通过 → 步骤二：识别资产属性 → 步骤三：打分
    ↓ 否决 → 输出🔴极度高危，终止
    ↓
步骤四：输出信号与投研建议
```

---

## 步骤零：标准信息接收

必须确认已获取：
- 代币名称 (Ticker) & 项目名称
- 所属公链 (Chain)
- 合约地址 (Contract Address) —— 核心锚点
- MC（流通市值）与 FDV（全流通市值）—— 用于计算流通率

---

## 步骤一：一票否决排雷（Kill Switch）

> ⚠️ **在计算任何看多得分前，必须先完成排雷。** 触发任意一条，立即终止评估，输出"🔴极度高危/拒绝买入"。

### 🔴 红线1：老鼠仓与发币者声誉
- **工具**：`okx-dex-trenches`（`memepump token-dev-info`）
- **否决条件**：
  - 开发者有直接Rug历史
  - Bundle Detection捆绑占比 > 30%
  - **新增**：历史Rug率 = rug_count / total_tokens > 10%（惯犯）
  - dev_reputation = "low" + bundle_ratio > 15%

**OKX实测数据解读**：
```
devLaunchedInfo.rugPullCount: 58
devLaunchedInfo.totalTokens: 1118
→ Rug率 = 58/1118 = 5.2% → 未超10%红线，但已属惯犯
```

### 🔴 红线2：流动性枯竭
- **工具**：`okx-dex-token`（`token holders`）
- **否决条件**：
  - 核心池子深度 < $50,000
  - 模拟大额交易滑点（Slippage）> 3%

### 🔴 红线3：蜜罐貔貅
- **工具**：OKX Security API / GoPlus
- **否决条件**：
  - 合约未开源
  - 包含隐藏Mint（增发）权限
  - 有黑名单功能
  - 无法卖出（疑似蜜罐）
  - 转账可被暂停

### 🔴 红线4：宏观环境熔断
- **工具**：DXY指数 + VIX
- **否决条件**：
  - BTC处于明确日线级别下跌通道
  - VIX > 25
- **原因**：此时山寨无独立行情，暂停看多操作

---

## 步骤二：识别资产属性与模型路由

获取数据：MC（流通市值）与FDV（全流通市值），计算**流通率 = MC / FDV**

| 流通率 | 模型 | 适用场景 |
|--------|------|---------|
| 15% - 80% | **模型A：常规模型** | 强叙事币 / VC币 |
| > 90% | **模型B：高流通/Meme模型** | 纯土狗 / Meme赛道 |
| < 15% | **模型C：数据可靠性低** | 预挖矿/极早期，数据可靠性低 |

## 步骤四：信号与投研建议

### 总分计算
```
总分 = 模块一得分 + 模块二得分
```

### 评分阈值（v1.1修复版）

| 总分 | 信号 | 结论 | 操作建议 |
|------|------|------|---------|
| **≥ 75分**（且未触发核心扣分项） | 🟢 强烈买入 | 强烈建议买入/建仓 | 筹码结构极佳，聪明钱流入+叙事爆发，高胜率做多标准。执行现货买入或合约右侧建仓。 |
| **55-74分** | ⚪ 底仓观望 | 底仓观望/等待突破 | 有一定叙事或数据亮点，但结构存在瑕疵（小额解锁/大盘无力）。可建观察仓跟踪，不符合重仓标准。 |
| **26-54分** | 🟡 风险提示 | 缺乏动能 | 缺乏增量资金，宏观信号偏弱，建议对冲或部分止盈。 |
| **≤ 25分** | 🔴 规避/减仓 | 规避/减仓/潜在做空标的 | 动能完全衰竭，抛压沉重。现货建议清仓；若标的在高位横盘且面临大额解锁/老鼠仓出货，转入【合约做空观察池】。 |

### 核心扣分项（触发即降级）
- 解锁抛压 -15（未来14天大额悬崖解锁）
- CEX资金流 -10（突发巨额净流入）
- 大户控盘 -15（占比>30%）
- 聪明钱砸盘 -20

## 数据获取优先级

1. **合约地址** → 链上数据（okx-dex系列）
2. **BTC.D + TOTAL3** → 判断大盘资金溢出方向
3. **MC/FDV** → 确定模型路由
4. **memepump token-dev-info** → 开发者Rug历史（红线1）
5. **security token-scan** → 蜜罐/合约安全（红线3）
6. **token holders** → 大户控盘度（模型B）
7. **解锁日历 + CEX资金流** → 判断抛压（模型A）
8. **Smart Money + Holder** → 判断庄家意图
9. **社交情绪** → 最后确认市场情绪
